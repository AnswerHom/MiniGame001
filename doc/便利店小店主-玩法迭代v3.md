## 二、顾客排队系统优化

### 2.1 排队逻辑增强

#### 当前问题
- 排队逻辑较简单，只显示等待状态
- 没有排队优先级的具体体现
- 顾客离开后，剩余顾客没有重新排序

#### 优化方案

**2.1.1 排队优先级系统**
```javascript
顾客优先级规则：
1. VIP顾客（👑）→ 优先级最高
2. 急切顾客（😰）→ 优先级中等
3. 普通顾客（😊）→ 优先级最低

实现方式：
- 顾客进入排队时分配优先级
- 服务时优先处理高优先级顾客
- VIP顾客插队逻辑
```

**2.1.2 智能排队管理**
```javascript
排队规则：
- 最多排队人数：8人
- VIP顾客可以插队到队伍最前面
- 排队时间过长（>8秒）的顾客会离开
- 空出位置后，后面顾客自动前移
```

**2.1.3 排队动画**
```css
排队动画效果：
- 顾客进入排队：从右侧滑入
- 顾客等待：轻微上下浮动
- VIP顾客：显示皇冠图标
- 等待时间：显示剩余秒数
```

### 2.2 顾客行为优化

#### 2.2.1 顾客购买决策
```javascript
购买逻辑：
1. 顾客扫描货架上的商品
2. 检查库存是否充足
3. 检查价格是否在可接受范围内
4. 如果满意 → 购买并离开
5. 如果不满意 → 离开（不排队）
```

#### 2.2.2 顾客满意度系统
```javascript
满意度计算：
基础满意度 = 价格合适度 × 库存充足度 × 服务速度

价格合适度：
- 价格太低（<最低价）：满意度 50%
- 价格合适（最低价~最高价）：满意度 100%
- 价格太高（>最高价）：满意度 30%

库存充足度：
- 库存充足（>0）：满意度 100%
- 库存不足：满意度 60%

服务速度：
- 收银台等级越高，服务速度越快
- VIP顾客对服务速度要求更高
```

#### 2.2.3 顾客忠诚度
```javascript
忠诚度系统：
- 第一次购买：忠诚度 0
- 满意度 > 80：忠诚度 +10
- 忠诚度达到 100：成为 VIP顾客
- 不满意：忠诚度 -20
- 长期不购买：忠诚度逐渐下降
```

## 五、技术实现要点

### 5.1 排队系统实现

#### 5.1.1 数据结构
```javascript
排队系统数据结构：
class QueueManager {
  constructor() {
    this.queue = []; // 排队顾客列表
    this.maxSize = 8; // 最大排队人数
  }

  // 添加顾客到排队
  addCustomer(customer) {
    if (this.queue.length >= this.maxSize) {
      return false; // 排队已满
    }

    // VIP顾客插队
    if (customer.priority === 3) {
      this.queue.unshift(customer);
    } else {
      this.queue.push(customer);
    }
    return true;
  }

  // 获取下一个服务的顾客
  getNextCustomer() {
    return this.queue.shift(); // 优先级高的在前
  }

  // 顾客离开
  removeCustomer(customerId) {
    this.queue = this.queue.filter(c => c.id !== customerId);
  }
}
```
